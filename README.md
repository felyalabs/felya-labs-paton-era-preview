# FELYA LABS Landing

Static Astro landing site for FELYA LABS.

The production output is generated by Astro into `dist/`. No production runtime server is required.

Initial page load is first-party only: local CSS, JavaScript, images, and fonts. There are no analytics, remote fonts, remote embeds, or third-party form requests before user action. Formspark is contacted only after a development-updates form submit; local prototype video media is requested only after the play cover is selected.

## Localized routes

The homepage is generated at `/en/`, `/de/`, `/ru/`, `/pt/`, `/fr/`, `/es/`, `/it/`, `/ky/`, `/id/`, `/ko/`, `/ja/`, `/zh-cn/`, and `/zh-tw/`. The root route detects a stored or browser language and redirects to the matching page; English is the fallback. Copy changes must be made in `src/i18n/translations.ts`, then checked with `npm run verify:i18n`. Run `npm run review:i18n` whenever review sheets need to be regenerated. After a build, `npm run verify:localized-build` checks localized routes, metadata, canonicals and hreflang links.

### Localization policy

- English is the canonical source for translation keys and source copy.
- Every localized dictionary is complete and independent. Localized dictionaries must not inherit values from another language.
- A change to one localized language must only change that language's dictionary and review sheet. For example, German copy must never update Russian website copy or the Russian review sheet.
- English copy changes do not silently overwrite existing translations. Each affected locale is reviewed and updated deliberately.
- New translation keys are added to English first and then explicitly translated for every supported locale. Verification fails while a locale is incomplete.
- Review sheets compare English directly with one target language and are not part of the deployed website.

## Deployment URL

Astro uses `https://preview.felyalabs.com` as the staging default. Set `SITE_URL` during a production build to generate canonical URLs, `hreflang`, Open Graph URLs, `robots.txt`, and `sitemap.xml` for another public origin.

## Local Development

The repository uses Bun exclusively. `package.json` pins the supported version and `bun.lock` is the authoritative lockfile.

```sh
bun --version
bun install --frozen-lockfile
bun run dev --host 127.0.0.1 --port 4321
bun run build
bun run verify
bun run preview --host 127.0.0.1 --port 4321
```
